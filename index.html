As We Wander

Link to Project

For a school assignment, I created a web blog that allowed users to sign in and add their own post. Using Google App Engine, I incorporated the ability for users to sign in with their google account, and saved their posts with Objectify and Datastore. I used basic HTML and CSS for the design, and deployed the website onto Google App Engine using Eclipse.

The theme of the blog followed travel posts, where users could share their experiences during their adventures. If they are not signed in with their google account, they can only read posts. The home page of the blog only shows the three latest posts, but a link on the bottom directs the user to another page to view all posts on the blog.

If the user chooses to add a post, the site directs them to sign in. Once signed in, the bottom of the page will include a text-input box for them to add a post. In addition to reading and adding posts, the user can also choose to subscribe to the blog, where they can receive a daily email containing the latest activity on the blog.

SureWalk

For a semester long group project, my team created an android app for our University's SureWalk Program, which allows students to request volunteers to safely walk them home late at night. Our app was designed for two types of users, students who were requesting to be walked home and volunteers who were accepting walk requests. My team spent the beginning of the semester creating mockups for the app, designing use case and class diagrams, and incorporating design patterns to organize how our code would be written.

Using Android Studio, we worked together to build each page of the app. We incorporated Google Maps API to allow the app to obtain the locations that students would be walking to. Since users can login and create new acounts, we saved user information and location data using Firebase API. After creating the application, we tested different user scenarios using Junit test cases.

ESW Conference App

Note: This is a project I am currently working on, and all information will be updated as progress is made.

Engineers for a Sustainable World (ESW) is a non-profit organization comprised of students, university faculty, and professionals who are dedicated to building a more sustainable world through various technical projects and outreach activities. Each year they hold a conference to bring together chapters from around the world. This year, the conference will be held in Austin TX and will consist of various events and keynote speakers. As a new feature, ESW hopes to include a mobile app that will allow attendees to stay organized and informed during the conference.

As a member of the ESW chapter at The University of Texas in Austin, I have taken on the task to design and build this app for the conference to use. Because the event will be held in April 2017, the following descriptions and images will show my plan and current progress of the app.

After sitting down with members of ESW headquarters, the following requirements were decided:

  • Users should see the schedule for the conference
  • Users should see the different options for speakers to attend during each session
  • Users should have access to a map of the area
  • Users can view suggested activities and food for the Austin area
  • The app should be available for both android and iOS

From these requirements, I created a preliminary design of the app that include four main pages: About Austin, Schedule, Sessions, and Map

Since the app needed to be available for both android and iOS, I decided it would be easier if I could develop using HTML, CSS, and Javascript. Buildfire, a platform for building mobile apps, allows developers to build customized plugins and pages using web technologies, while they take care of sending the app to Apple and Google Play. This will allow me to concentrate on the user design and functionality of the app, while Buildfire can verify that our app will be properly published.

Below are images of the schedule page of the app where users can view the events of the conference, and click each event to read more information. Everything below the Schedule header is developed with HTML, CSS, and Javascript, while the header and menu is part of Buildfire's platform.

Buildfire also includes a database to store user information and a global positioning system, that can be incorporated to improve our app's functionality with additional features. Future work will include devloping the remaining pages of the app and if time allows, using the datastore to allow the user to save events to their own schedule and using the GPS to give user's directions to specific events.
